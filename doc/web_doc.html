    4、Presale：(原名叫ICO)是代币预售配置入口，
    预售界面字段说明：
    Presale Assets
    • Total supply(RWAT) ：代币预售数量。系统获取到发行总量（Total supply）的x%作为默认数据带入，x%是后台对所有资产的配置项目，是第一优先级别
    • 调整滑杆：用户使用滑动条可以自定义预售数量
    • Bootstrap Liquidity(BEAC):引导流动性。引导流动性要求存入BEAC，没有BEAC的可以存入USDT或者去SWAP页面交换。这个字段数值是系统计算出来：计算的公式是：代币单价*预售数量。（代币单价是资产代币化价值除以代币发行总量，前者单位是USDT，后者是RWAT，即：USDT/RWAR）
    • Presale period（days）:预售期限，单位：天，后台对每一项资产配置。
    • Issue price：就是房产代币的单价，单位是：USDT/RWAT

5、Distribute：发行方存入资金分配给持币者入口。发行方理论上要按照确定的数额存入资金，如果存入数额不对，系统同样接受，差额部份平台线下和发行方交涉或者平台方垫付存入。如果发行方不存入资金，平台方存入发通知给持币者领取。资金数额后台针对每一项资产维护进系统。

  • Current holders:当前代币持有者（系统只统计在预售环节中购买了代币的人）
  • Holdings / Total:持有股份。这是指所有投资者当前持有代币的总数量占总发行量的比例，系统统计所有投资者钱包里的代币并求和并与总发行量对比计算这个数值。（分配收益就按照不同投资者钱包里持有的代币数量计算收益分成，应为目前收益不确定，分成有平台在适当的时候分配并发送通知，持币投资者自己领取）
  • Required Deposit(USDT):这是发行方为分配收益必须存入平台的资金数量，这个数量由后台针对每一项资产维护进系统。前端显示给发行方。
  • 如果发行方存入数量不足，也可以提交，平台线下催交。（发行方不分配的话，平台兜底，从后台存入这个资金，通知持币投资者领取）。
    6、Repurchas:发行方回购代币存入资金的入口。存入资金的数额由后台维护进系统并发送通知给发行方。回购价格按照发行价格（项目清盘下架的情形）。

关于资金的流向和定义：
1、Withdraw：铸造的代币发行方可以提现到钱包，用withdraw功能，全部数量提现，收取USDT作为铸造费，不分批提现，发行方是做市商
2、Propele:使用预售功能页提交代币预售信息，第一优先级别系统默认最少X%总数代币上预售市场，X%后台对所有资产统一配置，在pressale提交页的引导流动性默认BEAC，提交后入BEAC金库或者给交易机器人（吕工确定），如果用户选择USDT入USDT金库，在预售环节，避免房产代币全部卖光机器人缺少RWAT组流动性，预售环节增加了平台购买TWAT交给机器人的环节，平台工作人员人工转给机器人账户。
3、撮合交易市场使用BEAC作为流动性进行交易，交易对是RWAT/BEAC。用户的撮合市场交易账户可以充值：RWAT代币；BEAC，USDT，三个充值分别入合约资金库，用户可以从撮合市场账户里提现RWAT代币；BEAC，USDT；4、机器人交易说明：机器人控制大部分的房产代币，发行方控制小部分房产代币，即使发行方在撮合交易市场上全部出售的房产代币，套现的资金额远远小于其缴纳的引导流动性。机器人挂卖单的价格要克设置，卖单数量可以设置，交易频率可以设置。#### 初始机器人交易
机器人交易策略配置：为每一项资产配置交易机器人，配置项目有挂单价格梯度和数量，吃单价格梯度和数量，撤单价格梯度和数量，交易频率，交易深度`

3、My Investment字段和数据功能

这是投资者的投资清单（列表）入口，注意：这个数据列表投资者数据是全部投资数据，不区分投资是preasle market还是open market
（1）列表的字段和数据描述

  • Symbol:代币符号,数据来源于代币铸造
  • Asset Name:代币名称，数据来源于代币铸造
  • Shares Held（RWAT）:这是统计投资者购买代币存放在钱包里
    的数量，是钱包累积，计算收益分配比例使用这个数值
  • % held: 投资者钱包持有代币的比例，是钱包的累积，计算收益分配使用这个数据。
  • Spent(USDT):投资某一项资产总共花费的USDT,这个数据对应于投资者钱包里持有代币的数量计算。
  • clain earnings(BEAC):这是投资者领取的房产租金收益，用BEAC，
  • Redemption(USDT):这是投资者赎回自己的投资数量，用USDT。注意计算投资者赎回投资所一句的代币数量以钱包内持有的代币数量为一举计算。
  • Redemption Date：投资者赎回投资的日期
  • Contract：这是资产的合约地址，便于用户点击跳转到区块链浏览器查看
    （2）Action：操作
  • Claim:领取按键，包含两个子操作菜单

Tokens
Earnings

Tokens:这是给投资者领取预售代币的入口，如果预售未到期或者投资者没有参与预售，这个按键置灰色不可操作预售期限到期，投资者领回代币（RWAT**，星号表示编号）。
Earnings:这是投资者领取房产租金收益的入口，这个功能按钮置灰色，只有平台从后台存入资金，触发通知公告，持币者按照持币股份领取租金收益，持币者领取的Earnings使用BEAC。

  • Redeem:投资者赎回投资的入口，允许投资者钱包里持有的代币参与赎回。赎回是项目清盘下架的情形，这个按键是否可用分以下情形：

（1）代币预售期限未到期内购买的代币，不允许投资者赎回，这是第一优先级
（2）投资者在预售市场和二级开放市场购买的代币是否可以赎回，由后台控制开关状态决定（平台根据实际情况用开关控制是否允许投资者赎回自己的投资）这个开关加在每一项资产上面独立控制该资产的赎回权限
（3）赎回的价格：是代币预售时的发行价格。投资者赎回投资的币种采用USDT
